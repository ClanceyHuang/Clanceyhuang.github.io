Redis部署

2019-06-20 ops Redis 0 Comments Word Count: 872(words) Read Count: 3(minutes)

有时候很多的想法需要验证，只能在自己的试验机上部署对应的环境，这次讲讲怎么搞Redis环境。

安装Redis需要知道自己需要哪个版本，有针对性的安装，比如需要Redis GEO这个地理集合的特性，那么版本就不能低于3.2。另外需要注意的是版本号，小数点后是偶数代表稳定版本，单数代表非稳定版本。

  • 下载安装包
    到http://download.redis.io/releases/自行寻找自己需要的版本，我这里使用的4.0.14版本。
  • 解压安装包并安装

redis没有其他外部依赖，安装过程很简单。编译后在redis源代码目录的src文件夹中可以找到若干个可执行程序，安装完后，在/usr/local/bin目录中可以找到刚刚安装的redis可执行文件。

当Redis收到SHUTDOWN命令后，会先断开所有客户端连接，然后根据配置执行持久化，最后完成退出。
Redis可以妥善处理SIGTERM信号，所以使用kill redis进程的pid也可以正常结束Redis，效果与发送SHUTDOWN命令一样。
